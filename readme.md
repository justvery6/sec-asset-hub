🛡️ 灵析 · 安全运营资产管理平台使用手册
版本： V1.0 (稳定版)

定位： 企业级、轻量化、物理隔离的安全资产运营基石

## 一、 系统快速上手
### 1.1 访问与登录

服务地址： http://[服务器IP]:8080
默认账号信息：
超级管理员： admin / 123456（拥有系统全权限）
演示用户： test / 123456（属于基础架构组，仅拥有本组权限）
安全提示： 系统采用 Bcrypt 强哈希算法存储密码，确保数据库即使泄露也无法还原明文密码。

### 1.2 物理级界面隔离

本系统实现了真正的物理隔离逻辑：
管理员视角： 自动显示“团队管理”、“账号权限”菜单，右上角显示红色「系统管理员」标识。
普通操作员视角： 仅可见“现网资产”、“历史归档”及“系统公告”，无法通过任何前端手段强制打开管理页面。

## 二、 资产台账运营 (核心功能)
### 2.1 资产精细化登记

在点击“新增资产”时，系统支持两类靶标的差异化维护：
主机/IP 资产： 重点填写 IP 地址与端口。支持填写端口范围（如 80, 443, 8000-8080）。
Web/URL 资产： 重点填写完整路径（如 https://api.example.com）。
科学维度： 必须选定“运行环境”（PRD/UAT/DEV）与“数据敏感级”（L1-L4），以便后续安全分级保护。

### 2.2 批量资产导入

入口： 现网资产台账 -> 📥 批量导入。
模板格式： 需提供 CSV 文件，表头顺序必须为：
资产名称,资产类型,IP地址,URL路径,端口,公网暴露(是/否),环境,数据敏感级,归属团队,所属项目,用途说明,负责人
安全特性： 普通操作员导入资产时，系统会自动强行将“归属团队”修正为该用户所在团队，防止越权干扰。

### 2.3 资产下线与历史库

软删除机制： 点击“下线”并非物理删除，资产会消失在现网列表并同步移入「历史退役归档」。
溯源审计： 历史库记录了资产下线的准确时间，支持导出，满足合规审计需求。

## 三、 管理员高级维护
### 3.1 团队与账号分配
团队管理： 按业务线或组织架构建立团队（如：安全运营组、研发一组）。
账号管理： 管理员可以创建新用户，并将其指派给特定团队。指派后，该用户登录系统只能看到并操作所属团队的资产。

### 3.2 动态系统公告
全员广播： 管理员发布公告时不勾选接收人，系统内所有用户均可见。
定向下发： 在勾选框中选择特定团队的操作员，实现“点对点”下发高危漏洞排查通知或专项任务。

### 3.3 态势感知大屏
管理员首页自动呈现三大统计维度：
团队资产分布图： 监控各业务线资产体量。
资产类型比例图： 掌握 IP 与 URL 的数量分布。
7日变更趋势图： 洞察台账更新频率，监控录入进度。

## 四、 技术规格与安全性
后端驱动： Golang 1.22+
数据库： SQLite 3 (单文件存储，备份仅需拷贝 assets.db)
鉴权机制： JWT (JSON Web Token)，默认有效期 24 小时。
日志系统： * 开发模式： GORM Silent 模式，保持控制台清爽。
安全审计： 记录非法的未授权尝试与密码错误行为。
并发支持： 采用纯 Go 版本的 SQLite 驱动，无 CGO 依赖，支持高并发查询。

## 五、 维护说明

### 5.1 数据备份
由于系统采用无外部依赖架构，您只需每天定期备份程序目录下的 assets.db 文件，即可完成全量数据备份。

### 5.2 忘记密码
若超级管理员 admin 密码丢失，需使用数据库管理工具打开 assets.db，在 users 表中手动重置哈希值，或删除该文件重启程序重新初始化。
